# Contributing

This repo is primarily a personal learning log, but contributions are welcome — fixes, better explanations, additional exercises, or new mini-projects.

## How to contribute

1. Fork the repo
2. Create a feature branch: `git checkout -b feature/add-decorators-examples`
3. Make your changes, following the [style guide](./docs/style-guide.md)
4. Commit using [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`, `docs:`, `chore:`
5. Push and open a Pull Request using the PR template

## Guidelines
- Keep one topic/concept per file
- Include type hints and docstrings
- If adding a project, include its own `README.md`
- If adding an exercise, include both the problem and a working solution
- Run `black .` and `flake8` before committing if you have them installed

## Reporting issues
Use the issue templates in `.github/ISSUE_TEMPLATE/` for bugs or feature requests.

Thanks for contributing! 🎉
